wxPython From Scratch¶

wxPython; my way

Why I am writing this blog¶

I taught myself to write gui applications in wxPython; a sometimes painful experience.I wished I’d have had this guide by my side when I started. It would have save many hours of trial and error. Here, I want to share with others what little wisdom I have amassed in the process.

Why choose wx python?¶

I assume that you are already committed to using python to develop applications and that you now need or want to develop gui applications. There is a “native” python gui framework, tkinter, but on my initial investigation I felt that wxPython looked better. I freely admit that I might have been wrong about that. That said, wxPython does have all of the necessary widgets that I want and, in my opinion, looks great on linux, Windows and OSX. Prerequisites

I am assuming that you have downloaded wxPython and that you have a working knowledge of Python; in particular Object programming techniques including class definition, inheritance and Python’s treatment for a function or method call as an object.

Getting started¶

I’ve already used the “technical” term widgets so lets’ define a few terms used by wxPython:

widget: an element of gui programming, like a label, a button or a text box; frame: the rectangular window that contains all of the gui elements, (in fact it is usually called a window outside of wxPython (see this wiki article)). It usually has a title bar with control buttons, an area to display panels, and optionally a menu bar, a tool bar and a status bar; panel: an area on the frame that contains widgets. For most simple gui applications you will have one panel and it will cover the whole frame.

Fig 1. A basic frame

Figure 1. show a wxPython frame with a title bar. It has no menu bar, tool bar, status bar, nor any widgets. Before we look at how this was constructed, let’s first consider the wxPython main loop.

The wxPython main loop¶

A moments thought will convince you that a gui application must run some sort of loop. Most of the time, the screen sits there waiting for the user to do something. The application will stay in this loop, reacting to user events, until it is closed down.

The most basic code for a wxPython loop is:

import wx screen_app = wx.App() main_frame = MainFrame() screen_app.MainLoop()

in the first line we import the wxPython module; then, we create am instance of a wx application (wx.App); thirdly, we create an instance of a wx Frame (see below); and finally we start the wx main loop (app.MainLoop).

These four lines sum up the whole application; all of the complexity and wizardry is involved in creating the frame.

The wxPython Frame¶

In this section we will see how to create the minimal frame shown above. Later articles will take this simple basis and build more complex solutions.

The following code is sufficient to create the frame shown above. Remember, we have called the frame MainFrame and so we create a class of that name that inherits the class wx.Frame:

class MainFrame(wx.Frame):
def __init__(self):
wx.Frame.__init__(self, None, title=”Demonstrate wxPython Frame”) self.Show()

In this snippet, we have:

defined the class MainFrame which inherits wx.Frame; defined the dunder-init method for the class; called the wx.Frame dunder-init method to ensure that the frame is correctly initialised; called the Show method for the frame.

And that’s it. In eight lines of code we have created a frame and displayed it. It can be maximised, minimised and closed using the control buttons. Note that the frame picks up the theme that you have selected for your system (or uses the default if it has not been changed).

Summary¶

The most basic basic element of a gui program running wxPython is the Frame. A frame is created by instantiating an instance of a wx.Frame class within the wxPython main loop and calling its Show method. The full code is presented below:

import wx class MainFrame(wx.Frame): def __init__(self): wx.Frame.__init__(self, None, title=”Demonstrate wxPython Frame”) self.Show()

if __name__ == “__main__”: screen_app = wx.App() main_frame = MainFrame() screen_app.MainLoop()
